## Overview

**Project Name:** Syncloud  
**Type:** API Infrastructure  
**Technologies Used:** Node.js, Express, MongoDB  
**Live Platform:** [https://syncloud.vercel.app](https://syncloud.vercel.app)  
**GitHub repository:** [https://github.com/ikyawthetpaing/syncloud](https://github.com/ikyawthetpaing/syncloud)

Syncloud is dedicated to developing and maintaining the API infrastructure that serves as the backbone of our platform. With a strong focus on scalability, performance, and user-friendly interactions, Syncloud enables seamless communication between clients and our backend services.

## Technologies Used

Syncloud leverages the following technologies to provide a robust and efficient API infrastructure:

- **Node.js:** A JavaScript runtime built on Chrome's V8 JavaScript engine, enabling server-side JavaScript execution.
- **Express:** A web application framework for Node.js that simplifies the creation of robust APIs.
- **MongoDB:** A NoSQL database that provides flexibility and scalability for storing and retrieving data.

## API Documentation

### List of Countries

- **Endpoint:** `/api/v1/countries`
- **Method:** GET
- **Query Parameters:**
  - `fields`: Comma-separated list of fields to include in the response.
- **Example Requests:**

```http
GET /api/v1/countries
GET /api/v1/countries?fields=name,iso2,flag,states.name,states.state_code,states.cities.name
```

### Country by ISO2 Code

- **Endpoint:** `/api/v1/countries/:code`
- **Method:** GET
- **Path Parameter:**
  - `code`: ISO2 code of the country.
- **Query Parameters:**
  - `fields`: Comma-separated list of fields to include in the response.
- **Example Requests:**

```http
GET /api/v1/countries/US`
GET /api/v1/countries/US?fields=name,iso2,flag,states.name,states.state_code,states.cities.name
```
